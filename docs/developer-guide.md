# Developer Guide

This guide is for maintaining OPJRD as a browser-first experiment application
with a Tauri desktop wrapper. User-facing experiment setup is covered in
`docs/user-guide.md`.

## Prerequisites

OPJRD currently expects:

- Node.js `>=22.12.0`
- npm
- Rust/Cargo for Tauri development
- macOS Rust targets for universal macOS packaging:
  - `aarch64-apple-darwin`
  - `x86_64-apple-darwin`

Do not copy private support materials, participant data, secrets, or local
machine paths into the public repository.

## Repository Layout

- `src/`: browser-first TypeScript experiment core, rendering, adapters, and UI
- `src-tauri/`: Tauri shell, desktop commands, packaging config, and app icons
- `public/assets/examples/`: public fixture experiments
- `public/locales/`: bundled locale JSON files
- `docs/`: user, developer, config, output, compatibility, and packaging docs
- `tests/`: automated tests and golden-output fixtures
- `scripts/`: project maintenance scripts

## Architecture Boundaries

The experiment core should remain browser-first. Core modules should own config
normalisation, geometry, trial timing, scoring, output shaping, and the
jsPsych-facing trial flow.

Runtime-specific behaviour should stay behind adapters:

- browser and Tauri fullscreen handling
- browser-download and Tauri-local save behaviour
- Tauri config/file loading and folder access
- future JATOS save and metadata integration

The Tauri shell should provide desktop capabilities without making the trial
core depend on Tauri APIs. Browser mode must remain a first-class execution
path.

## Common Commands

Install dependencies:

```sh
npm install
```

Run browser development mode:

```sh
npm run dev
```

Run Tauri development mode:

```sh
npm run tauri:dev
```

Run tests:

```sh
npm test
```

Run the production browser build:

```sh
npm run build
```

Run linting and the privacy scan:

```sh
npm run lint
```

Run only the privacy scan:

```sh
npm run privacy:scan
```

Build the universal macOS app:

```sh
npm run tauri:build:mac
```

Build the universal macOS app, Tauri DMG bundle, and DMG SHA-256 checksum:

```sh
npm run tauri:build:mac:dmg
```

Release DMG signing, notarisation, stapling, and Gatekeeper checks are
maintainer release tasks that require the relevant Apple Developer credentials
and signing identities.

## Config and Editor Validation

`config.json` is the canonical settings model. The GUI settings editor must
write the same JSON model rather than introducing a separate configuration
format.

Experiment runs use strict config loading and should fail before trials begin
when invalid settings are present. The Tauri editor may use tolerant draft
loading so recoverable field-level problems can be fixed in the editor, but
saving must still run strict validation.

The locale list in the GUI editor is generated by the Vite plugin in
`vite.config.ts`, which scans `public/locales/*.json` at build/dev-server time.

## Data Contracts

JSON is the canonical output. CSV is a wide one-row-per-trial secondary export.

When changing output fields:

- update `docs/output-schema.md`
- update or add focused tests
- update golden fixtures only when the output contract intentionally changes
- keep participant metadata collision handling intact

Golden fixtures live under:

```text
tests/fixtures/golden/
```

## Stimuli and Assets

Text labels are the default stimulus rendering mode. Optional local image
stimuli are resolved relative to the experiment config folder in Tauri local
workflows and relative to the config URL in browser workflows.

Supported stimulus image extensions are `.svg`, `.png`, `.jpg`, `.jpeg`,
`.webp`, `.gif`, and `.bmp`. Add new formats only if they are reliably handled
in the supported browser and Tauri targets.

The app icon source is:

```text
src-tauri/icons/opjrd-icon.svg
```

Tauri's icon generator can regenerate desktop icon assets from that SVG. OPJRD
does not target mobile apps, so generated Android and iOS icon directories are
not kept.

## Release Hygiene

Before a public release candidate:

- run `npm test`
- run `npm run build`
- run `npm run lint`
- run `npm run privacy:scan` if lint was not run
- smoke-test supported browser and Tauri workflows
- build the universal macOS app
- sign, notarise, staple, and Gatekeeper-test the release DMG
- confirm public docs do not claim untested platform support
- confirm third-party notices and provenance notes are current

The fuller maintainer checklist is in `docs/release-checklist.md`.

JATOS should remain internal until its config, save, metadata, and deployment
path are minimally functional and documented.
